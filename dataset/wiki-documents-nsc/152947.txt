<doc id="152947" url="https://th.wikipedia.org/wiki?curid=152947" title="พีซี-บีเอสดี">พีซี-บีเอสดี พีซี-บีเอสดี (PC-BSD) เป็นเหมือนกับ Unix ตัวหนึ่ง เป็นระบบปฏิบัติการแบบ คอมพิวเตอร์แบบตั้งโต๊ะ ที่มีพื้นฐานบน FreeBSD ซึ่งคล้ายกับ DesktopBSD มันมีจุดมุ่งหมายเพื่อให้ง่ายต่อการติดตั้ง โดยใช้กราฟิกในการติดตั้งโปรแกรม และง่ายต่อการใช้งาน โดยมีค่าพื้นฐานมาจากระบบปฏิบัติการ KDE ก่อนการติดตั้งมีการติดต่อกับผู้ใช้เป็นระบบกราฟิก PC-BSD ยังเตรียมไดร์ฟเวอร์ของ nVidia ไว้เพื่อ ความง่ายและรวดเร็วของฮาร์ดแวร์ และมีข้อกำหนดในการติดต่อเป็นแบบ 3D Desktop Project PC-BSD ในปัจจุบันเป็นซอฟต์แวร์ติดตั้งแบบกราฟิกที่กำลังพัฒนา ซึ่งจะติดตั้งเป็นแบบ package ในเดือน สิงหาคม 2006 มันถูกโหวดให้เป็น สุดยอดระบบปฏิบัติการที่เป็นมิตรกับผู้เริ่มต้น โดยเวป OSWeekly.com ตั้งแต่ วันที่ 10 ตุลาคม 2006 iXsystems เป็นผู้สนับสนุน ในเรื่องการแก้ปัญหาฮาร์ดแวร์ให้กับ PC-BSD ในเดือน พฤศจิกายน 2007 iXsystems ได้มีข้อตกลงที่จะให้แจกจ่ายให้ Fry's Electronics ได้มีส่วนร่วมกับ PC-BSD เวอร์ชัน 1.4 ในเดือน มกราคม 2008 iXsystems ได้มีข้อตกลงที่คล้ายๆกัน ให้กับ Micro Centerประวัติ ประวัติ. FreeBSD ได้รวมรวบเอาระบบปฏิบัติการต่างๆที่มีไว้ใน PC-BSDการจัดการ package การจัดการ package. การจัดการ package ของ PC-BSD ได้หยิบเอาความต่างที่เข้าใกล้กับการติดตั้งระบบปฏิบัติการ Unix อื่นๆ แทนการใช้ประโยชน์ FreeBSD port หรือ ระบบ package PC-BSD ใช้ไฟล์ ชนิด .txz เมื่อคุณ ดับเบิ้ลคลิ้ก จะมีหน้าต่างแสดงขึ้นมาให้คุณติดตั้ง ซอฟต์แวร์ package และ Library ทั้งหมดจะถูกติดตั้งเองใน directory ชื่อ /usr/local เพื่อลดความสับสนว่า Binary Program อยู่ที่ใด และลดความเป็นไปได้ที่จะเกิดความเสียหายกับ system library เมื่อถูก upgrade หรือ เปลี่ยนแปลง การจัดการ package ของ PC-BSD ยังใส่ใจในเรื่องของการสร้างลิงก์ใน KDE menu บน KDE desktop ในปัจจุบันผู้พัฒนา PC-BSD กำลังพัฒนา ระบบ txz แบบใหม่ ซึ่ง รวบรวม track FreeBSD และกำเนิด txz ใหม่จาก port เหล่านั้น โดยอัตโนมัติ ให้เป็นประจำ โครงการ PC-BSD ได้ถูกเรียกร้องให้เป็นแบบของการจัดการ package ซึ่ง จะคล้ายๆกับ ระบบปฏิบัติการหลักของ Microsoft Windows จะใช้ง่ายกว่าระบบปฏิบัติการ Unix ตัวอื่นๆการอนุญาต การอนุญาต. ตั้งแต่มีการขอ License ก็ได้มีความขัดแย้งในเรื่องของอุดมการณ์ระหว่าง General Public License (GPL) กับ BSD License จุดมุ่งหมายทั่วไปจากการประเมินงานของค่าย BSD เกี่ยวกับ PC-BSD ต้นฉบับได้มี code ที่อยู่ภายใต้ License ของ GPL ขณะที่ Open Source BSD ที่สืบทอด License ส่วนมาก ไม่ได้รวมอยู่ภายใต้ BSD License GPL ต้นฉบับถูกใช้เพราะ PC-BSD project ใช้ QT toolkit จากการพัฒนาส่วนติดต่อ และผู้พัฒนาอยู่ภายใต้ ความติดใจที่จะใช้ application นั้น QT toolkit จะต้อง License ภายใต้ GPL หรือ QPL อย่างไรก็ตาม นี่คือสิ่งที่ไม่ถูกต้อง และ PC-BSD project หลังจากที่ License ใหม่อีกครั้ง code นี้ก็อยู่ภายใต้ BSD License</doc>
